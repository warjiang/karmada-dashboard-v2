# Product

## Register

product

## Users
Platform engineers and SREs operating Karmada-based multi-cluster environments. They use the dashboard to monitor cluster status, manage cross-cluster resources, apply and troubleshoot policies, and keep operations stable during daily changes and incident response.

## Product Purpose
Karmada Dashboard provides a practical control surface for multi-cluster operations. It helps operators complete common control-plane tasks quickly, with less context switching and lower cognitive load than raw API or CLI-only workflows. Success means faster operational actions, fewer configuration mistakes, and clearer cluster-wide visibility.

## Brand Personality
- Multi-cluster
- Easy-to-use
- Simple

## Anti-references
- Over-decorated marketing-style dashboards that prioritize visual effects over operational clarity.
- Dense legacy enterprise consoles with noisy layouts, unclear information hierarchy, and excessive interaction steps.

## Design Principles
- Prioritize operator workflows: critical actions and status signals must be easy to find and quick to execute.
- Keep complexity progressive: default views should be simple, advanced options should appear when needed.
- Make multi-cluster context explicit: always preserve clear cluster and scope awareness.
- Reduce operational risk: design for safe actions, predictable outcomes, and readable feedback.
- Optimize for repeated use: patterns should stay consistent across pages to reduce learning cost.

## Accessibility & Inclusion
Target WCAG 2.1 AA as the baseline. Maintain strong color contrast for status-critical UI, support keyboard navigation for core workflows, provide clear focus states, and avoid motion-heavy transitions that can distract during operations.
