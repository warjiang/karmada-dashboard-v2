# Karmada Dashboard v2 · 三方向共同设计 Spec

## 顾问式理解

这次重设计的本质不是“把 Ant Design 换一套皮肤”，而是把一个已经拥有大量 Kubernetes / Karmada 能力、但信息架构仍偏功能清单式的控制面，重组为运维人员能在压力下快速判断和行动的工作系统。核心用户包括平台工程师、SRE、多集群管理员和需要查看交付状态的研发负责人；他们打开 Dashboard 时最想先知道的是“全局是否健康、风险在哪里、哪一个集群或策略正在影响工作负载、下一步能安全做什么”，而不是浏览一组相互独立的菜单。新方案需要统一顶层多云视角与成员集群视角，减少上下文切换，同时保留资源表格、YAML 编辑、终端和指标探索等专业能力。静态交付必须覆盖正常、加载、空、部分失败、完全失败、无权限和危险操作确认等状态，数据可以 mock，但字段、术语和行为必须从真实项目长出来。基于这个理解，我直接做 3 个不同方向的真实版本给你看。

## 产品、受众与场景

- 产品：Karmada Dashboard v2 静态高保真原型。
- 主要受众：平台工程师、SRE、多集群管理员；次要受众为研发负责人和只读观察者。
- 核心场景：晨间巡检、异常定位、跨集群资源查找、策略影响确认、单集群下钻、容量检查、配置变更与回滚前确认。
- 输出：三方向各一个完整 HTML 初稿，统一按桌面工作台 `1440×900` 设计并响应到 `1280×720`；用户选定后扩展为全量静态页面组。

## 内容结构

1. 全局状态：Karmada control plane、成员集群健康、节点/CPU/内存/Pod 容量。
2. 风险与注意事项：失败、退化、未调度、策略冲突、指标缺失；必须可定位到对象。
3. 联邦拓扑：control plane、成员集群、策略与工作负载之间的关系。
4. 资源工作台：Namespace、Workload、Service、Config、Policy 的统一筛选、批量选择和详情下钻。
5. 成员集群：Overview、Workload、Service、Config & Storage、Cluster/RBAC。
6. 可观测性：控制面指标、组件筛选、时间范围、无数据与数据源异常状态。
7. 系统入口：终端、AI Assistant、语言、账号、版本信息，但不抢占主要任务层级。

## 状态矩阵

- Normal：有真实内容与健康状态。
- Loading：保留页面骨架和上下文，不用全屏阻塞 Spinner。
- Empty：说明“为什么为空”和下一步，例如更换 Namespace 或创建资源。
- Partial failure：数据面板可用，失败模块原位重试并标明影响范围。
- Fatal error：保留导航与诊断信息，提供返回、重试、复制错误详情。
- Forbidden：解释权限不足和所需 RBAC，不伪装成空数据。
- Destructive confirmation：展示对象、作用域、依赖关系和不可逆后果。
- Stale / disconnected：明确最后更新时间，禁止把旧数据继续显示成实时正常。

## 视觉与交互约束

- 延续 Karmada 官方帆船蓝，不重画 logo；语义色只用于状态。
- 正文 ≥14px、标签 ≥12px、可点击目标 ≥44px，正文对比度 ≥4.5:1。
- 使用 1.2 的 UI 字号音阶；中文行高 1.65；表格数字使用等宽数字。
- 导航必须同时回答“我在哪个层级”和“当前作用域是什么”，成员集群下钻不能像进入另一个产品。
- 支持全局搜索 / 命令入口，但初稿只做静态交互模拟。
- 关键动作靠文本和位置表达，图标只作为辅助；避免装饰性图标和无意义 KPI。

## 三个方向的隔离锚点

- 方向 A · 秒数轮盘：第 11 号 `Dark Editorial`。深墨工作台、左侧信号轨、单一青蓝强调；强调连续监控和快速异常扫描。
- 方向 B · 现实参照：Grafana 式观测工作台的“查询—时间—面板”结构，迁移为 Karmada 的 scope-first 控制画布；亮色、顶部作用域、中心画布、右侧事件流。
- 方向 C · 顶级定制：以 IBM Carbon 设计团队处理复杂企业工具的原则为锚，做“Federation Atlas”；黑白钴蓝、分层导航、地图与资源清单并置，强调可解释性和高风险操作秩序。

## Form 推导五问

- 叙事角色：初稿是总览 / 运维决策页，不是营销 hero。
- 观众距离：1 米内桌面显示器，支持长时间使用和高密度扫描。
- 视觉温度：冷静、权威、警觉；告警区域短促而不煽情。
- 容量估算：首屏包含 1 个全局结论、3 个集群、4 个容量信号、1 个风险列表，避免同质 KPI 卡超过 4 个。
- 视觉母题：联邦航线——Karmada 帆船的航行含义转译为跨集群连接、方向和状态，不把 logo 形状直接铺满背景。
